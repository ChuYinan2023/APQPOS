APQP-OS 技术概览

系统定位

APQP-OS 是一个 AI 驱动的 APQP 流程自动化系统。它将客户 RFQ 文件作为输入，沿着 18 个信息节点自动推导，最终输出完整的报价包和交付文件。

系统不替代工程师的判断——它负责信息提取、结构化、推导和一致性维护，工程师负责确认和决策。

技术架构

系统基于大语言模型（LLM）构建，结合汽车行业 APQP 领域知识深度定制，形成完整的智能工程系统。与通用 AI 工具有本质区别——这不是一个聊天助手，而是一套有严格工程纪律的自动化流水线。

核心技术栈：

技术层作用说明
Workflow 引擎流程编排18 个信息节点按有向无环图（DAG）自动调度，依赖关系自动管理，支持并行执行
Skill 系统领域知识封装每个节点的行业知识（如 DFMEA 评分规则、工艺模板、成本公式）以独立技能包形式封装，可扩展、可迭代
Agent 协作智能执行多个 AI Agent 并行处理不同文件和任务，自动分工、汇总结果，大幅提升处理速度
自动编排调度级联更新任意节点数据变更时，系统自动计算影响范围，按拓扑顺序逐级更新下游节点，确保数据一致性
置信度传播数据质量追踪每个数据点标注来源和可信度（S1-S5），低置信度自动向下游传播，报价依据透明可审计
人机协作控制决策边界AI 能确定的自动处理，不确定的强制暂停等待人工确认——系统不替人做判断，不悄悄猜测

与通用 AI 工具的区别：

  • 通用 AI 是"你问我答"——本系统是"按 APQP 标准自动推导，人只确认关键决策"
  • 通用 AI 输出不可追溯——本系统每个数字都能追溯到客户文件的具体章节
  • 通用 AI 无法保证一致性——本系统一处变更全程同步，不会有文件间的矛盾
  • 系统能力随使用持续积累，项目越多、数据越丰富，推导越精准

核心功能

1. 客户文件智能解析

输入：客户发来的 RFQ 文件包（PDF、Excel、Word、PPT，通常 5-20 个文件）

系统做什么：

  • 自动识别文件角色（采购规范、性能标准、技术规范、交付物清单、模板文件）
  • 递归提取嵌入文件（OEM 文件常有多层嵌套，如 Excel 里嵌 Word，Word 里嵌 PPT）
  • 从每个文件中提取结构化数据：性能要求、测试项、特殊特性、引用标准、几何参数
  • 识别客户提供的模板文件，自动关联到对应交付物

输出：结构化需求数据 + 数据缺口清单 + 文件索引

2. 设计要求结构化

输入：提取的客户需求数据

系统做什么：

  • 将每条客户需求翻译为供应商侧的设计目标
  • 按 8 个工程类别分类：功能性能、结构完整性、环境耐受、尺寸符合性、材料合规、电气/EMC、质量可靠性、外观标识
  • 量化每条指标，无法量化的标注"待工程师确认"
  • 检测不同文件间的数据冲突

输出：分类量化的工程指标表（DRG）

3. 产品结构与 BOM

输入：客户图纸/3D 模型 + 技术规范

系统做什么：

  • 生成组件树（零件清单、数量、材料提示）
  • 判断每个组件的 make/buy 决策：有明确依据的自动判定，不确定的标注出来让工程师确认
  • 有 3D 模型时精确提取几何参数，无模型时从图片和规范推导并标注假设

输出：多层级 BOM + 组件几何参数

4. 风险分析（DFMEA）

输入：组件树 + 装配接口 + 工程指标

系统做什么：

  • 按组件类型自动识别潜在失效模式（如管路→爆裂/泄漏/疲劳，接头→脱出/密封失效）
  • 按 AIAG 标准评分：严重度 × 发生频率 × 检测能力 = 风险优先级
  • 高风险项自动关联检测方法和推荐措施
  • 特殊特性（CC/SC）强制覆盖检查

输出：DFMEA 表 + 高风险项清单

5. 工艺路线与产能

输入：BOM（make 零件）+ DFMEA（高风险项需要的过程控制）

系统做什么：

  • 按产品类型匹配工艺模板（管路→挤出+弯管+装配，注塑件→模具+注射+检查）
  • 估算每道工序的设备投资、节拍时间、操作人数
  • DFMEA 高风险项自动在工艺路线中关联检测工序
  • 计算产线总投资、瓶颈工序、年产能

输出：工艺路线表 + 产能分析

6. 成本推算

输入：材料选型 + 用量 + 工艺路线

系统做什么：

  • 材料费：每个零件的重量 × 材料单价
  • 加工费：每道工序的人工 + 设备折旧 + 管理费
  • 一次性费用：模具投资 + 测试费 + 工程开发费
  • 汇总为单件成本（RC）和一次性费用（NRC）
  • 每个数字标注来源和可信度

输出：成本明细 + 单件报价 + NRC 报价

7. 交付物自动生成

输入：各环节产出的数据 + 客户模板文件

系统做什么：

  • 识别当前阶段需要提交的交付物清单
  • 优先使用客户提供的模板格式，自动填入数据
  • 无客户模板时使用公司标准格式或从零生成

输出：按客户格式填写好的交付文件（Excel/Word）

8. 变更级联更新

输入：任何环节的数据更新（如客户补了 3D 模型、确认了年产量）

系统做什么：

  • 自动计算受影响的下游环节
  • 只更新受影响的部分，不全量重跑
  • 更新后的交付文件自动重新生成

输出：更新后的全套数据和文件，变更记录可追溯

数据流

总体数据流

系统分四层处理，数据从上往下逐层推导：

层级处理内容输入输出
文件解析层PDF/Excel/PPT 递归提取嵌入文件，结构化客户 RFQ 文件包性能要求、测试项、特殊特性、引用标准、交付物清单
工程分析层设计要求(DRG) + 产品结构(BOM) + 风险分析(DFMEA) → 材料选型 + 用量计算 + 工艺路线文件解析层产出量化的工程指标、组件清单、失效模式、工艺方案
成本推算层材料费 + 加工费 + 管理费 → 单件成本(RC)；模具 + 测试 + 工程开发 → 一次性费用(NRC)工程分析层产出单件报价 + NRC 报价
报价输出层汇总所有数据，按客户模板生成交付文件成本推算层产出可提交的报价包（所有数据标注来源和可信度）

单个信息节点的处理流程

每个节点按统一流程执行，确保数据质量和可追溯性：

步骤做什么说明
1. 前置条件检查检查上游数据是否就绪有阻塞性缺口则暂停
2. 执行核心逻辑AI 推理 + 确定性计算不确定的标注待确认，不猜测
3. 写入结构化数据输出 JSON 格式数据每个数据点标注来源和可信度
4. 生成交付文件如果本节点负责某个交付物用客户模板填入数据，输出 Excel
5. 记录与报告技术审计日志 + 工作总结确保可追溯
6. 传递下游下游节点自动消费本节点产出DAG 驱动，无需手动传递

变更级联数据流

当任何环节的数据发生变更时（例如客户补充了 3D 模型），系统自动处理：

步骤动作示例
1. 更新源节点用新数据替换旧数据，提升可信度几何参数从假设值(S4) → 实测值(S1)
2. 计算影响范围自动分析哪些下游节点受影响BOM、用量、DFMEA、工艺、成本
3. 智能判断更新方式每个受影响节点：局部更新 or 全量重算材料不变(跳过)、用量重算、评分更新
4. 重新生成交付物所有受影响的交付文件自动重新生成更新后的 Excel 自动覆盖旧版本

数据可信度体系

系统中每一个数据点都标注来源和可信度等级：

等级来源含义示例
S1客户文件/法规硬性要求，不允许假设爆破压力 ≥ 36 bar（来自 PF 规范）
S2物理/几何计算数学推导管路净重 = π×(OD²-ID²)/4×L×ρ
S3公司历史数据来自内部数据库或经验人工工时费率 30 元/小时
S4行业基准必须标注为假设年产量 50,000（客户未说明）
S5AI 推断必须标注"待验证"管路总长度 ~600mm（从图片推测）

可信度向下传播：如果某个零件的长度是 S4（假设），那么基于它算出的重量、材料费、单件成本都继承 S4。报价文件中所有 S4/S5 项集中列出，提醒客户这些数据待确认。

多 OEM 适配

系统的业务逻辑与 OEM 特定格式解耦。核心引擎处理通用逻辑（组件类型→失效模式、工艺路线、材料推荐），OEM 适配层处理文件格式差异：

层级内容变更频率
核心引擎（通用）组件类型→失效模式映射、工艺模板、材料推荐、成本计算稳定，极少变更
OEM 适配层文件解析规则、术语映射、交付物模板格式每接入新 OEM 配置一次
OEM文件特点适配内容
StellantisSSTS + CTS + TDR + PF 规范，OLE 嵌入层层嵌套已完成
VW/AudiLastenheft + VW-Norm，德语文档，结构规范待配置
BMWQMT + BMW-Norm，PDF 为主待配置
通用 GMSSTS 类似但格式不同待配置

接入新 OEM 时，核心引擎不变，只需配置该 OEM 的文件解析规则和模板格式。

人机协作边界

系统自动做系统做不了（需要人）
文件解析和数据提取判断 RFQ 是否值得报价
结构化和分类确认 make/buy 决策（不确定的项）
失效模式识别和评分补充公司特有的工艺经验
成本计算和汇总确认利润率和报价策略
交付物生成最终报价审批
变更影响追踪与客户沟通确认缺口数据
数据一致性维护商务谈判

设计原则：系统能确定的自动处理，不确定的一定停下来让人确认。不会悄悄猜测。
