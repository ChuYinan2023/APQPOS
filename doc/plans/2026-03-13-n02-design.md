# n02 DRG 设计要求指南 — 设计文档

**日期**: 2026-03-13
**节点**: n02 — DRG 设计要求指南（Design Requirements Guide）
**类型**: auto（AI 全自动；描述性要求标注人工审阅）
**适用范围**: 任意汽车零部件 RFQ 项目，非特定产品类型

---

## 1. 定位与职责

n01 回答"客户说了什么"，n02 回答"我们要设计成什么"。

n02 消费 n01 的结构化字段（`performance_requirements`、`test_matrix`、
`special_characteristics`、`geometry`、`material_compliance`、`quality_targets`），
将规范条款翻译成**可量化、可测试、分功能分类的工程指标表**，供下游节点直接消费。

输出给：
- **n03** 3D 数模解析 — 几何与装配约束维度（CAT-04）
- **n07** DFMEA — 每个功能类别的设计目标，作为失效模式的评判基准
- **n09** DVP&R — 每条指标的测试方法和验收准则

## 2. 数据流

```
n01.payload
  ├── performance_requirements  ──┐
  ├── test_matrix               ──┤
  ├── special_characteristics   ──┤──► n02 ──► DRG 量化指标表
  ├── geometry                  ──┤

  └── quality_targets           ──┘

n07.dfmea_corrections（反馈边，可选）──► n02 修订版（drg_version + 1）
```

## 3. 功能分类（8 类，动态输出）

| ID | 类别 | 说明 |
|----|------|------|
| CAT-01 | 功能性能 | 零件主功能的核心性能指标，随产品类型不同而变化 |
| CAT-02 | 结构完整性 | 强度、爆破、拉脱、振动、疲劳——适用大多数零件 |
| CAT-03 | 环境耐受 | 温度范围、腐蚀、化学品暴露、湿度、UV |
| CAT-04 | 尺寸符合性 | 公差、GD&T、Cpk 目标，来自 geometry + SC/CC |
| CAT-05 | 材料与合规 | ELV/REACH/IMDS、禁用物质、可回收性 |
| CAT-06 | 电气/EMC | 电阻、ESD、EMI/EMC（仅当 n01 有相关要求时出现） |
| CAT-07 | 质量与可靠性 | 可靠性目标（P99C90/R95C90）、样品数量、GPAT |
| CAT-08 | 外观与标识 | 外观要求、零件标识、包装防护 |

> **动态原则**：n02 不预设零件"应有哪些类别"。某类别在 n01 中无对应条目时，
> 该类别不出现在输出 JSON 中。下游节点收到的每个 CAT 都有实质内容。

## 5. 审阅标注规则（三种触发条件）

| 触发条件 | `needs_review` | `review_reason` | 处理建议 |
|---------|---------------|-----------------|---------|
| 纯描述性要求，无数值可提取 | `true` | `"requirement_descriptive"` | 工程师补充具体目标值 |
| 同一参数两处来源数值不一致 | `true` | `"conflict"` | 工程师裁定以哪份文件为准 |
| 无原文依据，AI 推断填写 | `true` | `"low_confidence"` | 工程师确认或纠正推断值 |

所有 `needs_review: true` 的条目同步汇总到顶层 `review_items`，
方便工程师一次处理，无需遍历所有分类。

## 6. n07 反馈处理

n07 DFMEA 完成后若发现某功能 RPN 过高，可向 n02 发送修正请求：

1. `drg_version` 递增（1 → 2）
2. 对应 indicator 的 `design_target` 更新为更严格的目标
3. 原值保存在 `dfmea_correction.original_target`，新值在 `dfmea_correction.revised_target`
4. `dfmea_correction.gap_ref` 记录触发该修正的 n07 gap ID

```json
"dfmea_correction": {
  "gap_ref": "R-07-03",
  "original_target": "<原设计目标>",
  "revised_target": "<DFMEA 驱动的更严格目标>",
  "revised_at": "2026-03-15T10:00:00Z"
}
```

若 n07 新增了 n01 未覆盖的失效模式，n02 可在对应分类中新增 DRG 条目，
标注 `source_n01: "dfmea_feedback"`, `confidence: "S2"`。

## 7. Gap 规则

| Gap ID | 触发条件 | Severity |
|--------|---------|---------|
| R-02-01 | `performance_requirements` 为空，无法生成任何指标 | error |
| R-02-02 | 存在 `needs_review: true` 条目未经人工确认 | warning |
| R-02-03 | 存在 `conflict` 条目未解决 | warning |
| R-02-04 | CAT-04（尺寸）为空但 n01 有 geometry 字段 | warning |

## 9. 设计约束

- **不针对特定产品类型**：分类名称和 `design_target` 翻译逻辑均与产品无关
- **不硬编码标准号**：所有规范引用来自 n01 的 `referenced_standards`，不在 n02 中预设
- **不假设 n01 字段齐全**：每个字段读取前检查是否存在，缺失则标注 gap，不中断执行
- **n07 反馈是可选输入**：n02 首次运行不依赖 n07，反馈到来时执行增量更新
