# n03 3D 数模解析 — 设计文档

**日期**: 2026-03-14
**节点**: n03 — 3D 数模解析（3D Model Parser）
**类型**: auto-fallback（无 3D 模型时从 n01+n02 聚合；有 3D 模型时增量更新）
**适用范围**: 任意汽车零部件 RFQ 项目

---

## 1. 定位与职责

n01 回答"客户要求什么"，n02 回答"要设计成什么指标"，**n03 回答"物理结构是什么"**。

n03 输出**组件树**（零件级）和**装配接口表**（约束级），供下游节点直接消费：

```
n01.geometry
n01.special_characteristics（CC/SC 尺寸）  ──► n03 ──► components[] + assembly_interfaces[]
n01.material_compliance
n02.payload.categories[CAT-04]
```

输出给：
- **n04** BOM — `components[]`（子零件 ID、名称、类型、数量）
- **n05** 材料选型 — `components[].material_hint`
- **n06** 用量计算 — `components[].dimensions`（OD/ID/长度，计算重量/用料）
- **n07** DFMEA — `assembly_interfaces[]`（接口类型、SC/CC 引用、配合尺寸）
- **n08** 工艺路线 — `components[].type` + `dimensions`（决定加工工序）

---

## 2. 数据流

```
n01.payload
  ├── geometry              ──┐
  ├── special_characteristics ──┤── n03 聚合 ──► 组件树 + 接口表
  ├── material_compliance   ──┤
  └── quick_connector       ──┘

n02.payload.categories[CAT-04]  ──► 补充公差 / Cpk 目标

ext-3d（客户 Teamcenter）        ──► 到来时触发 geometry_version+1 增量更新（可选）
```

---

## 3. 运行模式

| 模式 | 触发条件 | has_3d_model | completeness_pct |
|------|---------|-------------|-----------------|
| **fallback** | 无 3D 模型，从 n01+n02 聚合 | false | 50–80%（视 n01 几何完整度） |
| **full** | 收到客户 3D 模型后增量补充 | true | 90–100% |

首次执行始终为 fallback 模式。`geometry_version` 从 1 开始，每次增量更新递增。

## 5. 组件识别规则（通用，不针对特定产品类型）

n03 根据 n01.geometry 字段语义**动态**识别组件类型，不硬编码零件名称：

| n01 字段模式 | 识别为组件类型 |
|------------|--------------|
| `*_od_mm` + `*_id_mm`（成对出现） | `tube` |
| `end_form.*` | `end_form` |
| `damper.*` | `damper` |
| `quick_connector.*` | `quick_connector` |
| `engine_hp_port.*` | 装配接口配合面（不单独成零件） |
| material_compliance 含 O-ring 材料 | `seal` |
| 无以上模式的其他 geometry 子节点 | `other` |

---

## 6. 装配接口识别规则

接口从以下信号推断：

1. **end_form 存在** → 管体与快接头之间存在 `quick_connect` 接口
2. **damper 存在** → 管体与减振器之间存在 `press_fit` 接口
3. **engine_hp_port 存在** → 管体与发动机 HP 口之间存在 `quick_connect` 接口
4. **SC/CC 含密封/拉脱力要求** → 对应接口标注 `sc_cc_refs`

---

## 7. Gap 规则

| Gap ID | 触发条件 | Severity |
|--------|---------|---------|
| R-03-01 | `n01.geometry` 为空，无法识别任何组件 | error |
| R-03-02 | `has_3d_model: false`（首次运行必触发） | warning |
| R-03-03 | 任意组件 `missing_dimensions` 非空 | warning |
| R-03-04 | 存在 SC/CC 尺寸指标但无对应接口记录 | warning |

## 8. 输出文件

---

## 9. 设计约束

- **不针对特定产品类型**：组件识别规则基于字段语义，不假设产品为燃油管或其他
- **不依赖 3D 模型文件**：fallback 模式下完全从 n01+n02 聚合，不解析 CAD 文件
- **n02 CAT-04 是补充源**：n01.geometry 为主，n02 CAT-04 补充公差和 Cpk 目标；若两者冲突，以 n02 为准（n02 已经过工程翻译）
- **增量更新路径**：3D 模型到来时，`geometry_version` 递增，只更新有变化的字段，`missing_from_3d` 条目逐条清除
