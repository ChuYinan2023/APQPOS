# 项目背景与目标

我们正在构建一个企业行为质量保障系统：把从客户需求到量产交付的完整信息流，变成一个有依据、可追溯、能协作的结构——从根源托底企业行为质量，从而确保企业长期利润目标。

从 RfQ 拿到手到产品 SOP 批量交付，这中间每一个信息节点的处理痕迹，都在这套系统里。

## 系统能力概述

汽车零部件从 RFQ 到量产，涉及大量跨部门、跨文件的信息处理。当前行业普遍面临的问题是：信息分散在各类文档中，人工传递容易遗漏，文件间的一致性难以保障，经验集中在个别人身上。

Q^AI 系统针对这一场景，将 AI 能力与 APQP 行业知识深度结合，实现以下核心能力：

**信息提取与结构化**
- 自动解析客户 RFQ 文件包，提取性能要求、测试项、特殊特性、引用标准、交付物清单
- 数据缺口第一时间识别并标注，按各 OEM 格式生成对应交付物

**跨环节数据一致性**
- 从需求到 BOM、DFMEA、工艺路线、成本推算，全链路数据自动关联
- 一处变更，所有相关环节和交付物同步更新，消除人工传递中的遗漏和矛盾

**风险与成本可推导**
- DFMEA 失效模式自动驱动 DVP&R 测试计划，覆盖关系可追溯
- 材料费、加工费、开发费逐项有推导依据，成本构成透明

**知识沉淀与复用**
- 项目执行过程中的数据、决策、经验沉淀在系统中
- 不依赖个人记忆，新项目可参考历史项目数据

目前第一阶段（报价自动化）已完成原型开发并通过实际项目验证，可提供 Demo 演示。

## 业务价值

- 报价快速精准——定点后不再系统性亏损
- 变更有完整记录——ECR/ECN 产生的成本影响有据可查，价格保护有理有据
- AI in loop，每一次信息脉动都有客观依据——从设计、验证到工业化全程数字化+极致精准高效
- 信息流完整，PPAP 首次通过率自然提高，量产节点不因返工延误，违约风险降低
- 核心知识沉淀在系统里，团队能力不依赖个人经验积累

## 落地路径

项目分两个阶段，6 个月内完成全部交付。

**第一阶段（3 个月）：报价阶段自动化**

拿到客户 RFQ 文件 → 系统自动完成需求解析、成本推算、交付物生成 → 输出可提交的报价包。分三步推进：

- **历史项目验证**：选 3-5 个已完成的不同品类历史 RFQ，用系统跑一遍，产出的报价与实际结果对标
- **实际项目并行**：选 2-3 个正在进行的 RFQ，系统和人工同时跑，对比结果和效率
- **正式使用**：团队独立使用系统处理新 RFQ

**第二阶段（3 个月）：全流程覆盖**

中标后的完整 APQP 流程——从计划定义到量产确认。报价阶段产出的 BOM、DFMEA、工艺路线等数据直接被后续阶段消费和精化，不重复录入。

## 第一阶段交付内容

系统在报价阶段覆盖的完整信息链：

| 环节 | 系统做什么 | 产出 |
|------|-----------|------|
| 客户文件解析 | 自动拆解 RFQ 文件包（PDF/Excel/PPT，含多层嵌入），提取性能要求、测试项、特殊特性、引用标准、交付物清单 | 结构化需求数据 + 缺口清单 |
| 设计要求指南 | 将客户需求翻译为可量化的工程设计目标，分 8 类，标注待确认项 | DRG 指标表 |
| 产品结构 | 生成组件树、装配接口、几何参数（有 3D 模型时精确提取，无模型时标注假设） | BOM + 组件清单 |
| 材料与用量 | 按组件类型推荐材料、计算重量和用量、验证合规性 | 材料选型 + 用量表 |
| 风险分析 | 按组件和接口识别失效模式，评分排序，高风险项自动关联检测方法 | DFMEA |
| 工艺与产能 | 按产品类型匹配工艺模板，估算设备投资、节拍、人员 | 工艺路线 + 产能分析 |
| 测试计划 | 从 DFMEA + 客户测试矩阵生成验证计划，确保高风险项全覆盖 | DVP&R |
| 成本推算 | 材料费 + 加工费 + 管理费 → 单件成本；模具 + 测试 + 工程费 → 一次性费用 | RC + NRC |
| 项目计划 | 按 APQP 阶段排里程碑，关联交付物和责任人 | 项目时间线 |
| 报价包 | 汇总以上所有数据，按客户模板生成交付文件，标注全部假设和待确认项 | 可提交的报价包 |

**关键设计原则：**

- 系统能确定的自动处理，不确定的一定标注出来让人确认——不会替人做判断
- 任何一处数据变更（如客户补了 3D 模型、确认了年产量），相关环节自动更新，不会有遗漏
- 所有数据标注来源和可信度（客户文件 > 公司经验 > 行业基准 > 待验证），报价依据透明

**验收标准：**

- 历史项目对标：成本偏差 ±15% 以内
- 效率提升：报价周期缩短 50% 以上
- 交付物质量：符合各 OEM 格式要求，报价工程师认可

## 第二阶段：全流程覆盖

报价阶段的数据不是用完就丢——中标后，BOM、DFMEA、工艺路线、测试计划直接进入后续阶段，逐步精化到量产级别。

| APQP 阶段 | 系统做什么 | 与报价阶段的关系 |
|-----------|-----------|----------------|
| 计划与定义 | 客户需求确认、可行性评审记录、项目团队与资源锁定 | 读取报价阶段的需求结构和项目计划 |
| 产品设计 | 设计评审跟踪、DFMEA 从初版精化到详细版、设计变更影响分析 | 报价阶段的失效模式 → 精化到完整版 |
| 过程设计 | PFMEA、控制计划、工装模具跟踪、产线布局 | 报价阶段的工艺路线 → 精化到工位级 |
| 产品与过程确认 | DV/PV 试验执行记录、PPAP 文件包生成、MSA/SPC | 报价阶段的 DVP&R → 关联实际试验结果 |
| 量产与持续改进 | 量产质量监控、经验教训沉淀、新项目复用 | 整个项目知识沉淀为下一个 RFQ 的参考 |

**核心价值：一条信息流贯穿始终。** 不是五个阶段五套文件，是同一份数据从粗到细、从假设到验证的演进过程。任何环节的变更都能追溯到源头，任何交付物都能推导出依据。

## 协作方式

| 阶段 | 我方 | 贵方 |
|------|------|------|
| 历史项目验证 | 部署系统、配置 OEM 规则、跑项目、对标分析 | 提供 3-5 个历史 RFQ 文件包和实际报价结果 |
| 实际项目并行 | 系统运行支持、问题响应、持续优化 | 指定 2-3 个在进行的 RFQ 项目，安排报价工程师配合 |
| 正式上线 | 培训、交接、运维支持 | 团队独立使用 |
| 第二阶段启动 | 需求调研、阶段设计、开发交付 | 提供全流程业务需求和历史项目数据 |

## 运维与支持

系统上线后的持续运维：

- 新 OEM 模板适配（客户拓展新客户时）
- 系统功能迭代与优化
- 使用问题响应
- 数据安全保障与备份

---

## 数据安全与保密

- 系统本地部署在贵方服务器，项目数据不出贵方网络
- AI 算力调用仅传输必要的文本信息，不传输客户原始图纸文件
- 所有项目数据按贵方现有信息安全管理制度管理
- 双方签署保密协议，我方对接触的贵方技术资料和商业数据承担保密义务

## 客户方其他投入

系统运行所需的服务器和算力资源按实际项目用量弹性配置：

| 项目 | 基础配置 | 扩展规则 | 首年估算 |
|------|---------|---------|---------|
| 服务器 | 1 台 | 项目并行量增加时按需扩展 | ~3 万元/年 |
| 网络 | 现有条件即可 | 需保障基本网络通畅 | — |
| AI 算力 | 按实际调用量 | 随项目数量线性增长 | ~3 万元/年 |
| 年度运维费 | 系统上线后每年 | 系统造价的 15%/年 | 按实际造价计算 |

以上服务器和算力费用按实收取，或由贵方自行支付。

**数字化基础建设**：系统落地可能涉及贵方在文件管理、数据规范等方面的数字化基础提升。我方将在第一阶段调研完成后，结合贵方现状给出具体的数字化建设方案建议，供贵方评估和决策。

**费用模式**：贵方作为首期合作伙伴，承担了前期开发投入和项目验证支持，以上资源费用仅按基础成本收取服务费，不含溢价。
